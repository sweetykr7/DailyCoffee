# 프로젝트 개요 — 데일리커피

> 핵심 요약 1페이지. 다른 건 몰라도 이것만 읽으면 됩니다.

---

## 목표

**coffeecg.com (커피창고)** 의 UI/UX를 최대한 동일하게 재현하되,
브랜드명만 **데일리커피 (Daily Coffee)** 로 변경한 실동작 커피 쇼핑몰 구축.

---

## 범위 (포함)

| 영역 | 내용 |
|------|------|
| 홈페이지 | 히어로 슬라이더, 카테고리 탭, 상품 섹션, 배너, 리뷰, 인스타, 푸터 |
| 상품 목록 | 카테고리별 필터링, 정렬(최신/인기/가격), 페이지네이션 |
| 상품 상세 | 이미지 갤러리, 옵션 선택(용량/분쇄도), 장바구니 담기, 리뷰 |
| 장바구니 | 수량 변경, 상품 삭제, 쿠폰 적용, 금액 계산 |
| 주문/결제 | 배송지 입력, 결제 요청(PG 모의), 주문완료 |
| 회원 | 회원가입, 로그인(JWT), 마이페이지 |
| 마이페이지 | 주문 내역, 리뷰 작성, 쿠폰/포인트, 주소록 |
| 리뷰 | 사진 첨부, 별점, 텍스트, 좋아요 |
| 관리자 | 상품 CRUD, 주문 상태 관리, 회원 목록 (기본) |
| 인프라 | Docker Compose (front + back + db), 환경별 .env |

## 범위 (제외)

- 실제 PG 연동 (토스/아임포트) — 모의 결제만
- 실시간 재고 관리
- 다국어 지원
- 모바일 앱

---

## 핵심 제약

- **토큰 효율**: MD 파일을 통해 컨텍스트 전달. 중복 설명 금지.
- **Docker 필수**: 로컬에서 `docker-compose up` 하나로 전체 실행
- **실동작**: 실제 DB에 데이터 저장/조회 되어야 함
- **브랜드**: UI 어디에도 "커피창고" 없음. 전부 "데일리커피" / "Daily Coffee"
